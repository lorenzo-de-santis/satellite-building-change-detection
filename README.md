# Satellite Building Change Detection on LEVIR-CD

Final clean repository for a deep learning project on binary building change detection from bitemporal satellite imagery.

The project compares three models under the same preprocessing, loss, training, and evaluation pipeline:

1. `simple_cnn`: shallow full-resolution convolutional baseline.
2. `unet`: early-fusion U-Net with a six-channel bitemporal input.
3. `siamese_temporal_attention_unet`: Siamese U-Net with shared temporal encoding and bottleneck temporal attention.

The final implementation removes exploratory hyperparameter-search scripts and keeps only the selected model configurations, reproducible runners, HPC scripts, result summaries, figures, and report sources.

## Final Results

All models were trained on aligned 256 x 256 LEVIR-CD patches and evaluated on the held-out test split with a fixed threshold of 0.5.

| Model | Best epoch | Epochs ran | Test F1 | Test IoU | Precision | Recall | Test loss |
|---|---:|---:|---:|---:|---:|---:|---:|
| Simple CNN | 120 | 139 | 0.7093 | 0.5496 | 0.5895 | 0.8903 | 0.5404 |
| U-Net | 39 | 51 | 0.8840 | 0.7922 | 0.8787 | 0.8895 | 0.1681 |
| Siamese Temporal Attention U-Net | 63 | 69 | 0.9019 | 0.8214 | 0.9000 | 0.9039 | 0.1189 |

The U-Net provides the largest improvement over the simple CNN baseline, while the Siamese Temporal Attention U-Net achieves the best final F1-score and IoU.

## Included Content

- Final model definitions only.
- Shared preprocessing and dataset loading.
- Weighted BCE + Dice loss.
- Metrics for F1, IoU, Dice, precision, recall, accuracy, and loss.
- Training runner with CPU, CUDA, and Apple MPS device detection.
- SLURM scripts for HPC execution.
- Final numeric results, histories, summaries, and report-ready figures.
- LaTeX paper source and images.

## Excluded Content

The dataset and trained `.pt` checkpoints are intentionally excluded from GitHub because they are large and should not be versioned directly. The final metrics and figures are included, while checkpoints can be regenerated by running the training script.

Ignored files include:

- `data/`
- `*.pt`, `*.pth`, `*.ckpt`
- Python caches
- LaTeX build artifacts
- raw logs and temporary HPC output

Each original 1024 x 1024 image pair is split into aligned 256 x 256 patches by the shared preprocessing pipeline.

## Installation

Create an environment and install the dependencies:

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

For CUDA training, install the PyTorch build matching the target CUDA version if the default package is not appropriate for the machine.

## Local Training

From the repository root:

```bash
python run_training.py \
  --data_root /path/to/LEVIR-CD \
  --output_dir results \
  --device auto \
  --num_workers 0
```

`--device auto` selects CUDA when available, then Apple MPS when available, otherwise CPU.

To train a subset of models:

```bash
python run_training.py \
  --data_root /path/to/LEVIR-CD \
  --models unet siamese_temporal_attention_unet \
  --output_dir results \
  --device auto
```

## HPC Training

From the repository root:

```bash
sbatch slurm/run_training.slurm
```

If the dataset is not in the default HPC path, pass `DATA_ROOT` explicitly:

```bash
DATA_ROOT=/path/to/LEVIR-CD sbatch slurm/run_training.slurm
```

## Figure Generation

After training:

```bash
python generate_figures.py \
  --data_root /path/to/LEVIR-CD \
  --output_dir results/figures \
  --simple_cnn_checkpoint results/<run>/simple_cnn/best.pt \
  --unet_checkpoint results/<run>/unet/best.pt \
  --siamese_temporal_attention_checkpoint results/<run>/siamese_temporal_attention_unet/best.pt \
  --simple_cnn_history results/<run>/simple_cnn/history.csv \
  --unet_history results/<run>/unet/history.csv \
  --siamese_temporal_attention_history results/<run>/siamese_temporal_attention_unet/history.csv
```

On HPC:

```bash
RUN_DIR=results/<run> sbatch slurm/generate_figures.slurm
```

Each figure is saved as a separate image: one graph, mask, heatmap, overlay, or error map per file.
